# 概述

## 需求背景

当前项目管理过程中，需求评估与成本管控环节存在诸多痛点，严重影响项目推进效率与成本可控性。在需求评估阶段，传统模式依赖人工对《产品需求说明书》进行拆解与评估，缺乏统一的标准化流程，导致不同评估人员的判断存在主观差异，评估结果科学性、合理性不足，且评估周期较长，无法快速支撑项目研发计划的制定。在成本管控阶段，项目经理需手动整理OA项目相关财务、人力信息，核算流程繁琐、数据滞后、易出错，且无法实现成本消耗的实时监控；同时，实际成本与计划成本的偏差难以实时捕捉，无法及时发现成本风险，易导致项目超支、延期。基于此，亟需搭建数字人系统，通过智能化技术替代人工繁琐操作，实现需求评估标准化、成本管控实时化、偏差监控可视化，破解当前项目管理中的核心痛点。

## 业务目标

1. 需求评估标准化：搭建数字人需求评估能力，实现《产品需求说明书》的智能化解析、拆解与评估，输出结构化的需求工作量评估明细表，统一评估标准，将需求评估周期缩短，提升评估结果的科学性与合理性。

2. 成本管控精细化：实现OA项目截图关键财务、人力信息的OCR自动识别，支持用户确认补充后，自动计算可消耗成本、日人力成本、可消耗天数及成本燃尽日期；支持人员入项、离项动态调整，调整后自动重新核算成本数据，确保成本核算精准、高效。

3. 偏差监控实时化：构建项目成本偏差分析机制，实时监控实际成本消耗情况，对比项目计划成本，精准识别偏差数据，通过图表展示等方式，让项目经理实时掌握项目成本健康度，及时发现成本风险。

4. 数据管理安全化：实现项目数据按用户隔离存储，保障不同项目经理的项目信息、成本数据、评估结果的安全性与独立性，避免数据泄露或混淆。

## 业务价值

1. 提升效率，降低人力成本：通过数字人智能化处理需求评估、OCR自动识别、成本自动核算等工作，替代传统人工繁琐操作，大幅缩短需求评估周期与成本核算时间，减少人工投入，降低人力成本。

2. 规范流程，提升管理质量：建立标准化的需求评估流程，消除人工评估的主观偏差，保障技术评估的科学性与合理性；通过精细化成本管控，规范成本核算流程，提升项目管理的标准化、规范化水平。

3. 精准管控，降低项目风险：实时监控项目成本消耗与偏差情况，及时预警成本超支风险，助力项目经理快速调整人力方案、优化成本分配，避免项目超支、延期，保障项目研发计划精准推进。

4. 赋能管理，支撑决策制定：为项目经理提供一体化的需求评估与成本管控工具，清晰呈现工作量、成本数据及偏差信息，为项目排期、人力调配、成本优化等决策提供精准的数据支撑，提升项目管理决策的科学性。

# 业务概述

## 业务概述 

本项目旨在搭建具备三项核心能力的数字人系统，聚焦项目全生命周期的工作量评估与成本管控，为项目经理提供一体化管理工具。数字人可接收《产品需求说明书》，通过标准化解析、拆解与评估，输出结构化工作量评估明细；同时依托OCR识别技术提取OA项目关键信息，实现项目成本实时核算、人力方案动态调整，以及实际成本与计划成本的偏差监控、可视化展示。系统以标准化、智能化为核心，打通需求评估与成本管控的全流程，解决传统管理模式中效率低、标准不统一、偏差难把控等痛点，助力项目研发计划精准落地与成本高效管控。

## 业务处理流程

本系统业务处理流程以项目经理操作全流程为核心，从登录验证到功能调用，逻辑清晰、步骤明确，具体流程如下：

1. 用户登录：项目经理通过系统指定的账号密码（或其他授权方式）登录系统，系统完成身份验证，确认用户权限后，允许用户进入系统首页；若身份验证失败，提示用户核对账号密码并重新登录，保障系统数据安全。

2.  进入首页：登录成功后，系统自动跳转至首页，首页是项目经理进入系统后的核心总览与操作入口，分为上方项目统计仪表盘与下方三大功能入口区。

3. 首页上方项目统计仪表盘展示：仪表盘仅展示当前用户所属项目数据，实时统计项目总量、成本健康状态与进度健康状态，异常数据标红高亮，帮助项目经理快速识别风险。

4. 首页下方功能区操作：首页下方设置三大核心功能区，项目经理可根据实际需求点击对应功能区，下方功能区提供实施成本预估、成本消耗预估、成本偏差监控快捷入口，支持一键跳转对应功能页面，从子页面返回时数据自动刷新，形成 “总览监控 — 快速操作” 的一体化业务闭环。

（1）实施成本预估：点击该功能区，进入成本预估页面，可上传《产品需求说明书》，由数字人完成智能化解析、拆解与评估，输出结构化的需求工作量评估明细表，同时可基于评估结果，完成项目实施阶段的成本预估核算。

上传 / 导入项目《需求规格说明书》，系统启动流程，开始读取需求文档内容。

模型基于需求文档，自动提取并梳理出项目完整的业务功能列表，完成需求的结构化拆解，对拆解出的功能点进行循环识别与估算，分析每个功能项复杂度基准、关联系统个数。

展示模型分析结果，用户确认功能列表及功能项复杂度基准、系统关联度系数；确认流程系数配置、技术栈系数、人天单价、管理系数。

系统根据功能项复杂度基准、系统关联度系数、流程系数配置、技术栈系数、人天单价、管理系数，计算单个功能项对应的项目阶段（需求、设计、开发、技术测试、性能测试、投产上线）的人天数。汇总所有功能项的估算结果，合计计算项目整体的产品、UI、研发、测试、项目管理总人天，完成各角色的工作量核算。

系统生成并输出可视化估算界面（含工作量明细、成本占比、合规校验、计算轨迹），以及标准工作量估算模板，支持项目经理导出、调整、归档。

成本消耗预估：点击该功能区，进入成本消耗预估页面，通过OCR识别功能上传OA项目截图，系统自动提取财务、人力等关键信息，经项目经理确认补充后，自动计算可消耗成本、日人力成本、可消耗天数及成本燃尽日期，支持人员入项、离项调整，调整后系统自动重新核算相关数据。

**图片上传**：上传 OA 项目信息截图（支持多张）。

**OCR 识别**：系统提取合同金额、售前比例、税率、外采金额、当前人力成本、成员列表。

**信息确认**：用户补充成员等级，系统自动返显对应日成本。

**自动核算**：计算可消耗成本、日人力成本、可消耗天数、燃尽日期。

**方案调整**：新增 / 移除成员、设置入项 / 离项时间，重新计算。

**数据保存**：项目信息持久化存储。

（3）成本偏差监控：点击该功能区，进入偏差监控页面，通过OCR识别功能上传OA项目截图，系统自动提取财务、人力等关键信息，经项目经理确认补充后，自动计算可消耗成本、日人力成本、可消耗天数及成本燃尽日期，支持人员入项、离项调整，调整后系统自动重新核算相关数据。

5. 流程闭环：项目经理完成对应功能操作后，可返回首页查看项目进展更新情况，根据项目实际需求，重复调用不同功能区，实现项目工作量评估、成本管控、偏差监控的全流程闭环管理。

### 业务功能：上传需求文档

#### 业务流程

数字员工接收用户发起的 IT 项目成本预估任务，进入文档上传环节。

数字员工引导用户提供符合要求的需求规格说明书（DOC/DOCX 格式）。

数字员工自动校验文档格式及内容完整性。

校验通过后，数字员工解析文档内容，提取功能需求、技术栈、业务流程等关键信息，为后续成本计算做准备；校验不通过则自动反馈问题并引导重新上传。

#### 业务功能详述

文档接收与校验：自动接收用户上传的文档，校验格式（仅支持DOC、DOCX），并检查需求说明书文档中的“第三章 功能描述->3.2.标准系统功能描述”中的每个功能项是否包含完整业务流程、功能详述、业务规则、输入和输出。“第三章 功能描述->3.3.特色系统需求”每项内容是否齐全，涉及即体现具体内容，不涉及则需写明“不涉及”、“暂无”不可以为空。“第四章 数据要求”及“第五章 非功能及系统级需求”涉及即体现具体内容，不涉及则需写明“不涉及”、“暂无”不可以为空。

文档解析与信息提取：自动解析文档内容，提取功能模块、技术选型、业务流程节点、非功能性需求等关键信息，结构化存储为计算数据源。

上传状态反馈：实时反馈文档上传进度、校验结果及解析状态，确保用户清晰掌握环节进展

#### 业务规则

文档格式必须为 DOC 或 DOCX，否则数字员工自动拒绝接收并提示修正。

文档需包含完整的功能需求描述、详细的业务流程说明、性能指标和非功能性需求否则视为无效文档，数字员工引导补充后重新上传。

文档解析失败时，数字员工自动重试 1 次，重试后仍失败则终止当前环节并反馈原因

#### 输入输出

输入：用户上传的需求规格说明书（PDF/DOC/DOCX 格式）。

### 业务功能：**成本核算（核心）****-****成本监控**

#### 业务流程

补充完成项目信息

系统成本和预期自动核算

图形化展示界面及调整建议

#### 业务功能详述

阶段成本核算公式

各角色成本 = ∑（各角色人员已报工时/8 ）* 单价

需求成本 = ∑产品经理角色成本

设计成本 = ∑UI设计成本 +∑（开发工程师成本 + 技术经理成本）× 0.3

开发成本 =∑（开发工程师成本 + 技术经理成本）× 0.7

技术测试成本 =∑ 测试工程师成本 × 0.7

性能测试成本 =∑ 测试工程师成本 × 0.3

投产成本 =∑（需求+设计+开发+技术测试+性能测试）成本 × 0.02

当前项目总成本AC = ∑所有成员成本之和

进度与偏差计算

任务完成进度（%）= 【用户输入】或【DevOps已完成任务数 / 总任务数 × 100%】

当前成本消耗占比（%）= 当前项目总成本 / （项目合同金额（1-预期利润比例））

成本偏差 = 实际成本消耗占比 - 标准阶段成本占比

各团队人员成本预期情况基于用户上传的工作量评估表

方案一：对存量项目进行成本偏差分析（当前方案）

产品团队预期成本：产品阶段工作量

UI团队实际成本：设计阶段工作量*0.5

研发团队实际成本：设计阶段工作量*0.5+研发阶段工作量

测试团队实际成本：技术测试阶段+性能测试阶段工作量

管理团队成本：投产阶段工作来那个

方案二：对使用数字员工小M进行工作量成本评估的项目

使用小M完成工作量评估后，数据信息做持久化存储

各阶段的预期使用成本直接从数据库查询关联带出

团队人员成本实际使用情况：

产品团队实际成本：∑产品经理角色成本

UI团队实际成本：∑UI角色成本

研发团队实际成本：∑（开发工程师、技术经理）角色成本

测试团队实际成本：∑测试工程师角色成本

管理团队成本：∑（项目经理+项目负责人）角色成本

#### 业务规则

利润空间：项目经理预期的利润空间，至少将税率、外包采购、售前、硬件设备等其他等成本包住

如果用户上传的是工作量评估表：标准阶段成本占比按照工作量评估表计算；
