### 1.7 核心数据流与计算规则

#### 工作量评估数据流
```
需求文档 → 【AI解析】→ 功能拆解 → 【参数配置】→ {复杂度基准、系数组} 
→ 【标准公式计算】→ 各阶段工作量 → 【团队成本映射】→ 各团队成本汇总
→ 【结构化输出】→ Excel报告
```

#### 成本消耗数据流
```
OA截图 → 【OCR识别】→ {合同金额、售前比、税率、人力成本}
→ 【信息确认】+ 【人员等级补充】→ 【4项核算公式】
→ {可消耗成本、日人力成本、可消耗天数、燃尽日期}
```

#### 成本偏差数据流
```
项目多源截图 → 【大模型识别】→ 项目信息+进度+成本
→ 【基准选择】(工作量评估表/默认比例) → 【阶段成本分解】
→ 【预期vs实际对比】→ 【偏差分析+AI建议】→ 可视化展示
```

## 二、系统整体架构与技术方案设计

### 2.1 技术栈选型方案

#### 前端技术栈
```
框架层：Vue 3.x + Element Plus
状态管理：Redux Toolkit
UI组件库：Ant Design 5.x（企业级规范）
图表库：ECharts 5.x（复杂数据可视化）
HTTP客户端：Axios
文件处理：xlsx（Excel生成）、crypto-js（加密）
表单验证：React Hook Form + Yup
工程化：Vite + ESLint + Prettier
```

**选型依据**：
- React：大型企业应用的事实标准，生态完整
- TypeScript：金融系统必选，类型安全
- Ant Design：国内金融企业规范，组件齐全
- ECharts：复杂图表需求（折线+柱形+饼图）
- Redux Toolkit：复杂状态管理，计算结果缓存

#### 后端技术栈
```
框架：Node.js + Express.js（或Java Spring Boot备选）
认证：JWT + 国密算法（SM4）
数据库：PostgreSQL 14+（金融级稳定性）
ORM：Sequelize / TypeORM
文件存储：本地加密存储（满足内网部署要求）
AI集成：调用AI中台RESTful API
缓存：Redis（计算结果、session）
日志：Winston（全量操作日志审计）
部署：Docker + Kubernetes（可选）
```

**选型依据**：
- PostgreSQL：国产金融系统标配，ACID保证
- JWT：无状态认证，适合分布式
- 国密SM4：符合网络安全等级保护三级
- Redis：提升OCR/大模型识别响应

#### 外部服务依赖
```
AI大模型：调用AI中台API
  - 需求文档解析（4.3.1）
  - 项目信息识别（4.5.1）
  - AI人员调整建议（4.5.4）

OCR服务：调用基础服务OCR引擎
  - 合同金额、税率等关键字段识别（4.4.1）
  - 支持多OCR引擎对比校验

OA系统：数据来源（用户截图）
  - 无需API对接，通过截图+OCR方式
```

### 2.3 数据库表结构大纲

#### 核心表设计

```sql
-- 1. 用户表（权限隔离基础）
users (user_id, username, password_hash, role, status, created_at)

-- 2. 项目基础信息表
projects (project_id, user_id, project_name, status, created_at, updated_at)

-- 3. 成本估算表（需求文档→工作量评估结果）
cost_estimates (
  estimate_id, project_id, doc_name, 
  estimate_params {complexity_base, system_coefficient, process_coefficient, ...},
  total_workdays, total_cost, teams_breakdown {product, ui, dev, test, pm},
  status, created_at
)

-- 4. 参数配置表（可复用模板）
parameter_templates (
  template_id, user_id, template_name,
  config {complexity_map, coefficient_rules, unit_price_map},
  is_default, created_at
)

-- 5. 成本消耗表（OA截图→核算结果）
cost_consumptions (
  consumption_id, project_id,
  contract_amount, pre_ratio, tax_rate, external_cost_labor, external_cost_software,
  current_cost, team_members {name, level, daily_cost, join_date, leave_date},
  available_cost, daily_labor_cost, available_days, burnout_date,
  created_at, updated_at
)

-- 6. 成本偏差监控表（实际vs预期）
cost_deviations (
  deviation_id, project_id,
  baseline_type, baseline_params,
  expected_stages {stage_name, expected_cost_ratio},
  actual_stages {stage_name, actual_cost, actual_cost_ratio},
  deviation_data {stage, deviation_value, variance_direction},
  ai_suggestions, created_at
)

-- 7. 操作日志表（审计&追溯）
audit_logs (
  log_id, user_id, project_id, action, details, ip_address, created_at
)
```

#### 关键字段说明
- **user_id**: 所有表的最高隔离级别，确保用户数据完全隔离
- **estimate_params/config**: JSON字段，存储动态参数配置
- **teams_breakdown**: JSON数组，{team_name, workdays, unit_price, total_cost}
- **team_members**: JSON数组，支持动态增删，自动按时间区间重算成本
- **created_at/updated_at**: 所有表标配，支持数据版本追溯

## 三、Claude Code分阶段开发实施方案

### 阶段规划原则
? **先基建后业务**：权限系统 → 通用组件 → 核心模块  
? **先框架后细节**：全量页面骨架 → 功能补全 → 交互优化  
? **先P0后P1**：MVP可用 → 增强功能 → 体验优化  
? **分阶段交付可验证**：每个阶段完成后可独立运行和测试

### 【阶段一】项目初始化与基础架构搭建（1-2天）

**核心目标**：完成前后端初始化、基础框架搭建、开发环境就绪

**前置条件**：获得PRD、技术栈确认、依赖服务API文档

**详细执行步骤**：

1. **前端初始化**
   - 用Vite创建React18+TypeScript项目
   - 配置Ant Design、ECharts、Redux Toolkit、Axios
   - 建立项目目录结构：src/{pages,components,services,store,utils,styles}
   - 配置ESLint/Prettier规范
   - 建立.env环境变量配置（API地址、加密密钥等）

2. **后端初始化**
   - 初始化Node.js + Express项目
   - 配置PostgreSQL连接池、Redis客户端
   - 建立项目目录：src/{routes,controllers,services,models,middleware,config}
   - 配置路由前缀（/api/v1）、全局错误处理、日志系统
   - 建立密钥管理（JWT密钥、数据加密密钥）

3. **公共基础建设**
   - 前后端通信协议定义（统一API response格式）
   - 认证流程设计（登录→token生成→权限校验→自动登出）
   - 数据隔离中间件实现（所有查询自动加where user_id=当前用户）
   - 加密工具函数（密码hash、数据加密/解密）

4. **数据库初始化**
   - 执行建表脚本（users、projects等7个核心表）
   - 建立数据库备份策略
   - 建立测试数据集（3个测试项目）

**阶段交付物**：
- ? 前端项目框架完整可运行（npm start）
- ? 后端API框架完整可运行（npm run dev）
- ? 数据库表及初始数据
- ? 前后端通信协议文档
- ? 开发者环境使用手册

**需求覆盖校验**：
- [ ] 权限隔离基础 (R-0101~0102)
- [ ] 数据库存储基础 (NF-41)

**验收标准**：
- 前端访问localhost:5173可看到项目壳
- 后端GET /api/v1/health返回200
- 数据库可正常连接且表结构完整

---

### 【阶段二】用户认证与权限体系开发（2-3天）

**核心目标**：完成登录功能、会话管理、权限隔离的全链路实现

**前置条件**：阶段一完成

**详细执行步骤**：

1. **后端认证服务**
   - 实现POST /api/v1/auth/login接口（校验账号密码、返回JWT token）
   - 实现JWT middleware（解析token、校验有效期、自动续期）
   - 实现密码策略（连续错误5次锁定15分钟）
   - 实现会话超时（30分钟无操作自动登出）
   - 实现操作日志记录（所有接口自动记录user_id、action、ip）

2. **前端登录页**
   - 页面layout：居中卡片式登录表单
   - 表单字段：用户名（必填）、密码（必填）、登录按钮
   - 表单校验：实时校验、错误提示（账号或密码错误时红色提示）
   - 成功处理：登录后存储token到localStorage、跳转首页
   - 失败处理：显示错误信息、清空密码字段

3. **权限隔离实现**
   - 前端路由守卫：未登录访问任何页面都重定向到login
   - 后端数据隔离：所有GET/POST/PUT/DELETE接口都自动加where user_id=当前用户
   - 实现权限检查中间件：检查当前用户是否有操作权限
   - 实现会话token中间件：每个请求都验证token有效性

```
7. AC（总成本）= Σ 所有成员成本
```

### 1. JWT 与密码加密工具（src/utils/crypto.ts）
```typescript
import jwt from 'jsonwebtoken';
import bcryptjs from 'bcryptjs';

// 密码加密（注册时）
export async function hashPassword(password: string): Promise<string> {
  const salt = await bcryptjs.genSalt(10);
  return bcryptjs.hash(password, salt);
}

// 密码校验（登录时）
export async function verifyPassword(password: string, hash: string): Promise<boolean> {
  return bcryptjs.compare(password, hash);
}

// 生成 JWT token
export function generateToken(userId: number, role: string): string {
  const token = jwt.sign(
    { userId, role },
    process.env.JWT_SECRET!,
    { expiresIn: '24h' }  // token 有效期 24 小时
  );
  return token;
}

// 校验与解析 JWT token
export function verifyToken(token: string): { userId: number; role: string } | null {
  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET!);
    return decoded as { userId: number; role: string };
  } catch (error) {
    return null;
  }
}
```

```typescript
// JWT 认证中间件
export function authMiddleware(req: Request, res: Response, next: NextFunction) {
  // 从 Authorization header 或 Cookie 中获取 token
  const token = req.headers.authorization?.split(' ')[1] || req.cookies.token;

  if (!token) {
    return res.status(401).json({ code: -1, message: '未授权，请先登录' });
  }

  const decoded = verifyToken(token);
  if (!decoded) {
    return res.status(401).json({ code: -1, message: 'Token 已过期或无效' });
  }

  // 将 userId 和 role 附加到 request 对象，后续中间件可直接使用
  req.userId = decoded.userId;
  req.userRole = decoded.role;

  next();
}

// 权限检查中间件（如果需要检查特定角色）
export function requireRole(...roles: string[]) {
  return (req: Request, res: Response, next: NextFunction) => {
    if (!roles.includes(req.userRole)) {
      return res.status(403).json({ code: -1, message: '权限不足' });
    }
    next();
  };
}
```

### 4. 账号锁定机制（src/utils/accountLock.ts）
```typescript
import redis from 'redis';

const client = redis.createClient({
  url: process.env.REDIS_URL!,
});

// 记录密码错误次数
export async function recordFailedLogin(username: string): Promise<number> {
  const key = `login_failed:${username}`;
  const count = await client.incr(key);
  
  // 第一次错误时设置过期时间为 15 分钟
  if (count === 1) {
    await client.expire(key, 900);  // 900 秒 = 15 分钟
  }
  
  return count;
}

// 检查账号是否被锁定
export async function isAccountLocked(username: string): Promise<boolean> {
  const key = `login_failed:${username}`;
  const count = await client.get(key);
  return count ? parseInt(count) >= 5 : false;
}

// 清除错误计数（成功登录后）
export async function clearFailedLogin(username: string): Promise<void> {
  const key = `login_failed:${username}`;
  await client.del(key);
}

// 获取剩余锁定时间（秒）
export async function getRemainingLockTime(username: string): Promise<number> {
  const key = `login_failed:${username}`;
  const ttl = await client.ttl(key);
  return ttl > 0 ? ttl : 0;
}
```

### 5. 登录路由（src/routes/auth.ts）
```typescript
import express, { Request, Response } from 'express';
import db from '../config/database';
import { hashPassword, verifyPassword, generateToken } from '../utils/crypto';
import { recordFailedLogin, isAccountLocked, clearFailedLogin, getRemainingLockTime } from '../utils/accountLock';
import { authMiddleware } from '../middleware/auth';

const router = express.Router();

// POST /api/v1/auth/login
router.post('/login', async (req: Request, res: Response) => {
  const { username, password } = req.body;

  // 输入校验
  if (!username || !password) {
    return res.status(400).json({ code: -1, message: '用户名和密码不能为空' });
  }

  // 检查账号是否被锁定
  const locked = await isAccountLocked(username);
  if (locked) {
    const remainingTime = await getRemainingLockTime(username);
    return res.status(429).json({
      code: -1,
      message: `账号已锁定，请在 ${remainingTime} 秒后重试`
    });
  }

  try {
    // 查询用户
    const result = await db.query('SELECT * FROM users WHERE username = $1', [username]);
    if (result.rows.length === 0) {
      // 记录错误次数
      const failCount = await recordFailedLogin(username);
      const message = failCount >= 5 ? '账号已锁定，请 15 分钟后重试' : '用户名或密码错误';
      return res.status(401).json({ code: -1, message });
    }

    const user = result.rows[0];

    // 密码校验
    const passwordMatch = await verifyPassword(password, user.password_hash);
    if (!passwordMatch) {
      // 记录错误次数
      const failCount = await recordFailedLogin(username);
      const message = failCount >= 5 ? '账号已锁定，请 15 分钟后重试' : '用户名或密码错误';
      return res.status(401).json({ code: -1, message });
    }

    // 登录成功，清除错误计数
    await clearFailedLogin(username);

    // 生成 JWT token
    const token = generateToken(user.user_id, user.role);

    // 将 token 存储在 HTTP Only Cookie 中（增强安全性）
    res.cookie('token', token, {
      httpOnly: true,
      secure: process.env.NODE_ENV === 'production',  // 生产环境下仅在 HTTPS 传输
      sameSite: 'strict',
      maxAge: 24 * 60 * 60 * 1000  // 24 小时
    });

    // 记录操作日志
    await db.query(
      'INSERT INTO audit_logs (user_id, action, details, ip_address) VALUES ($1, $2, $3, $4)',
      [user.user_id, 'LOGIN', JSON.stringify({ username }), req.ip]
    );

    res.json({
      code: 0,
      message: '登录成功',
      data: {
        token,
        user: {
          userId: user.user_id,
          username: user.username,
          role: user.role
        }
      }
    });
  } catch (error) {
    console.error('Login error:', error);
    res.status(500).json({ code: -1, message: '服务器错误' });
  }
});

// POST /api/v1/auth/logout
router.post('/logout', authMiddleware, async (req: Request, res: Response) => {
  try {
    // 清除 Cookie 中的 token
    res.clearCookie('token');

    // 记录操作日志
    await db.query(
      'INSERT INTO audit_logs (user_id, action, details) VALUES ($1, $2, $3)',
      [req.userId, 'LOGOUT', JSON.stringify({})]
    );

    res.json({ code: 0, message: '登出成功' });
  } catch (error) {
    console.error('Logout error:', error);
    res.status(500).json({ code: -1, message: '服务器错误' });
  }
});

export default router;
```

```typescript
export class BaseModel {
  protected table: string;

  constructor(table: string) {
    this.table = table;
  }

  // 查询单条记录
  async findById(id: number, userId: number) {
    const query = `SELECT * FROM ${this.table} WHERE id = $1 AND user_id = $2`;
    const result = await db.query(query, [id, userId]);
    return result.rows[0];
  }

  // 查询所有用户的记录
  async findByUserId(userId: number) {
    const query = `SELECT * FROM ${this.table} WHERE user_id = $1 ORDER BY created_at DESC`;
    const result = await db.query(query, [userId]);
    return result.rows;
  }

  // 通用插入（自动添加 user_id）
  async create(userId: number, data: any) {
    const columns = ['user_id', ...Object.keys(data)];
    const values = [userId, ...Object.values(data)];
    const placeholders = columns.map((_, i) => `$${i + 1}`).join(',');
    const query = `INSERT INTO ${this.table} (${columns.join(',')}) VALUES (${placeholders}) RETURNING *`;
    const result = await db.query(query, values);
    return result.rows[0];
  }

  // 通用更新（自动检查 user_id，确保用户只能更新自己的数据）
  async update(id: number, userId: number, data: any) {
    const updates = Object.keys(data).map((k, i) => `${k} = $${i + 2}`).join(',');
    const query = `UPDATE ${this.table} SET ${updates} WHERE id = $1 AND user_id = $${Object.keys(data).length + 2} RETURNING *`;
    const result = await db.query(query, [id, ...Object.values(data), userId]);
    return result.rows[0];
  }
}
```

#### 5.3.2 计算准确性验收

| 公式 | 验收标准 | 测试案例 |
|-----|--------|--------|
| **工作量评估** | 各阶段占比在合理区间（需求12-18%、设计12-20%等）? 流程系数自动调整有效? | 输入不同参数，验证占比调整 |
| **可消耗成本** | 公式：合同×(1-售前-税) - 外采人力 - 外采软件 - 当前成本 ? | 输入具体数字，手工计算验证 |
| **可消耗天数** | 公式：ROUND(可消耗成本 / 日人力成本, 0) ? 工作日计算剔除周末 ? | 测试跨周末日期计算 |
| **阶段成本** | 需求=Σ产品成本 ? 设计=ΣUI + Σ(开发+技术)×0.3 ? 开发=Σ(开发+技术)×0.7 ? 等 | 多个成员的团队，验证分配准确 |
| **偏差** | 偏差 = 实际消耗占比 - 标准阶段占比 ? | 已知预期和实际，验证偏差计算 |

### 6.2 关键决策与确认项

   - 建议：在阶段一完成后立即进行API联调测试

   - 本方案使用PostgreSQL + Redis
   - 如贵司已有其他数据库规范，请提前沟通

   - 本方案使用JWT + HttpOnly Cookie
   - 支持与贵司现有认证系统集成，需提供集成文档

   - 支持内网私有部署，满足安全合规要求
